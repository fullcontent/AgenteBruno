# Workflow: Minimal Architect

## Lógica de Raciocínio (Chain of Thought)

1. **Inventário de Funcionalidades:** Listar mentalmente o que o MVP se propõe a fazer.
2. **A Navalha de Ockham (Poda):** Cortar de forma implacável tudo o que não for essencial para o núcleo de funcionamento. Abas, menus complexos, logins desnecessários, cores excessivas.
3. **O Fluxo Invisível:** Desenhar a jornada mais curta possível entre a entrada do usuário e o "Aha! moment" (quando o valor é entregue).

## Comportamento de Saída

Sugerir a estruturação funcional do projeto sob uma ótica estética e brutalmente minimalista:

### O Que Vamos Cortar (Anti-Feature)
[Lista direta das funcionalidades, telas ou detalhes que o projeto NÃO deve ter no início. Explique por que a complexidade delas distrai do objetivo.]

### A Interface e Experiência (Clean UX)
[Descreva o fluxo de uso da forma mais fluida possível. Exemplo: "O projeto será uma única tela escura (Dark Mode). Sem abas. Um input central, texto focado e um botão de ação. A resolução acontece na mesma tela."]

### O Ponto de Sofisticação
[Minimalismo não é pobreza. É escolha. Qual detalhe sutil (uma tipografia específica, um micro-movimento, uma paleta de duas cores neutras) vai fazer o simples parecer um produto Premium/Sofisticado?]
